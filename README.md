Concesionario Coches

Requisitos mínimos - Necesario llegar al 4/6 para sumar adicionales

El programa debe gestionar 
• Coches que como mínimo tengan marca, modelo, matricula, precio, año y kilómetros
• Clientes con dni, nombre y teléfono
• Ventas con idVenta, cliente, coche, fecha y precio de venta.

Cuando se arranca el programa, este debe cargar como mínimo 5 coches y 2 clientes.
La aplicación, que ha de tener una interfaz de consola deberá permitir a los usuarios 
las siguientes opciones:

1. Añadir coches al concesionario
2. Mostar los coches disponibles
3. Buscar coches usando:
4. 
a. Marca
b. Rango de precios
c. Año

6. Registrar un nuevo cliente, mostrando un error si ya existe un cliente con el 
mismo dni
7. Registrar una venta, indicándole el cliente y el coche involucrados
8. Listar ventas
9. 
La interfaz deberá ser intuitiva y sencilla. Validará los datos necesarios (Tenéis que encontrar vosotros las validaciones necesarias)
Cumplir con los requisitos mínimos tendrá una nota máxima (Pudiendo sufrir deducciones durante la corrección de 6)

Requisitos adicionales
Filtros avanzados (1 pnt)
• Realizar búsquedas múltiples, pudiendo enviar varios.
• Ordenar coches por precio marca y año

Estadísticas del concesionario (2pnts)
Se podrán ver las estadísticas de un vendedor, mostrando:
• El precio medio de los coches que ha vendido
• El coche más caro que ha vendido
• Cuantos coches ha vendido 
o Cuantos coches ha vendido desglosado por marca
• Cuanto suman los coches vendidos

Uso MVC (1pnt)
• Añade un logo al concesionario empleando código ascii
• Crea tablas para separar la información 
• Crea una mejora de tu elección que facilite la visualización
• Se emplea una arquitectura MVC
